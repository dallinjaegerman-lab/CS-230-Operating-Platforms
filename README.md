# CS-230-Operating-Platforms
CS-230 class

The client, The Gaming Room, wanted to expand its existing Android game, *Draw It or Lose It*, into a web-based application that could support users on Windows, macOS, Linux, Android, and iOS. The software needed to be scalable, secure, and accessible through a web browser while supporting thousands of players. My goal was to design a solution that met these requirements while considering the advantages and limitations of each operating platform.

I think I did particularly well in evaluating the different operating systems and recommending the best platform for the project. I compared Linux, Windows, macOS, and mobile devices based on hosting capabilities, licensing costs, development tools, and client compatibility. This helped justify why Linux would be the most effective server platform while using responsive web technologies to support all client devices.

If I could revise one part of my work, I would expand the discussion of security and scalability. Although I addressed these topics, I could have learned more about these topics and included more detail about cloud hosting, load balancing, database optimization, and security practices to better demonstrate how the application could support future growth.

I interpreted the client's needs by focusing on creating a browser-based application that would work consistently across desktop and mobile platforms. Instead of designing separate applications for each operating system, I recommended using responsive web technologies to provide a consistent user experience while reducing development and maintenance costs. Considering the user's needs is essential because software is only successful if it is accessible, reliable, easy to use, and performs well on the devices that users already own.

When approaching the software design, I first analyzed the client's requirements, compared available technologies, and evaluated the strengths and weaknesses of each platform. I then selected the technologies that best met the project's goals while balancing cost, scalability, and maintainability. In future projects, I would continue using a structured design process that includes gathering requirements, creating architectural plans, evaluating technology options, identifying risks, and documenting decisions before beginning implementation. This approach helps produce software that is easier to develop, maintain, and scale over time.
